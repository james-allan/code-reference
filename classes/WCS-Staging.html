get_duplicate_site_lock_key()

woocommerce-subscriptions/vendor/subscriptions-core/includes/class-wcs-staging.php : 172

Generates a unique key based on the sites URL used to determine duplicate/staging sites.

public static get_duplicate_site_lock_key() : string

The key can not simply be the site URL, e.g. http://example.com, because some hosts (WP Engine) replaces all instances of the site URL in the database when creating a staging site. As a result, we obfuscate the URL by inserting '[wc_subscriptions_siteurl]' into the middle of it.

We don't use a hash because keeping the URL in the value allows for viewing and editing the URL directly in the database.

Gets the URL Subscriptions considers as the live site URL.

public static get_live_site_url([int|null $blog_id = null ][, string $path = '' ][, string|null $scheme = null ]) : mixed

This URL is set by @see WCS_Staging::set_duplicate_site_url_lock(). This function removes the obfuscation to get a raw URL.

Parameters
$blog_id : int|null = null

The blog to get the URL for. Optional. Default is null. Used for multisites only.

$path : string = ''

The URL path to append. Optional. Default is ''.

$scheme : string|null = null

The URL scheme passed to @see set_url_scheme(). Optional. Default is null which automatically returns the URL as https or http depending on @see is_ssl().

get_site_url_from_source()

woocommerce-subscriptions/vendor/subscriptions-core/includes/class-wcs-staging.php : 273

Gets the sites WordPress or Subscriptions URL.

public static get_site_url_from_source([string $source = 'current_wp_site' ]) : string

WordPress - This is typically the URL the current site is accessable via. Subscriptions is the URL Subscritpions considers to be the URL to process live payments on. It may differ to the WP URL if the site has moved.

Parameters
$source : string = 'current_wp_site'

The URL source to get. Optional. Takes values 'current_wp_site' or 'subscriptions_install'. Default is 'current_wp_site' - the URL WP considers to be the site's.
