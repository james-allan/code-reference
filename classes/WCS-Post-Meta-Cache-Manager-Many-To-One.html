in package
  • WooCommerce

woocommerce-subscriptions/vendor/subscriptions-core/includes/class-wcs-post-meta-cache-manager-many-to-one.php : 15

Class for managing caches of post meta data that have a many-to-one relationship, meaning only one cache should exist for the meta value. This differs to WCS_Post_Meta_Cache_Manager which allows multiple caches for the same meta value i.e. a many-to-many relationship.

WordPress has special handling for meta deletion on all posts rather than a specific post ID. This method handles that case.

is_change_to_ignore()

woocommerce-subscriptions/vendor/subscriptions-core/includes/class-wcs-post-meta-cache-manager.php : 68

Check if the post meta change is one to act on or ignore, based on the post type and meta key being changed.

protected is_change_to_ignore(int $post_id[, string $meta_key = '' ]) : bool

One gotcha here: the 'delete_post_metadata' hook can be triggered with a $post_id of null. This is done when meta is deleted by key (i.e. delete_post_meta_by_key()) or when meta is being deleted for a specific value for all posts (as done by wp_delete_attachment() to remove the attachment from posts). To handle these cases, we only check the post type when the $post_id is non-null.

Parameters
$post_id : int

The post the meta is being changed on.

$meta_key : string = ''

The post meta key being changed.

Return values
bool — False if the change should not be ignored, true otherwise.
