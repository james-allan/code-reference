calculate_next_payment_date()

woocommerce-subscriptions/vendor/subscriptions-core/includes/class-wc-subscriptions-order.php : 1985

Takes a subscription product's ID and calculates the date on which the next payment is due.

public static calculate_next_payment_date(mixed $order, int $product_id[, string $type = 'mysql' ][, mixed $from_date = '' ]) : mixed

Calculation is based on $from_date if specified, otherwise it will fall back to the last completed payment, the subscription's start time, or the current date/time, in that order.

The next payment date will occur after any free trial period and up to any expiration date.

Parameters
$order : mixed

A WC_Order object or the ID of the order which the subscription was purchased in.

$product_id : int

The product/post ID of the subscription

$type : string = 'mysql'

(optional) The format for the Either 'mysql' or 'timestamp'.

$from_date : mixed = ''

A MySQL formatted date/time string from which to calculate the next payment date, or empty (default), which will use the last payment on the subscription, or today's date/time if no previous payments have been made.

Tags
since
1.0
Return values
mixed — If there is no future payment set, returns 0, otherwise it will return a date of the next payment in the form specified by $type

get_matching_subscription()

woocommerce-subscriptions/vendor/subscriptions-core/includes/class-wc-subscriptions-order.php : 904

Get a subscription that has an item with the same product/variation ID as an order item, if any.

private static get_matching_subscription(WC_Order $order[, int $product_id = '' ]) : null|object

In Subscriptions v1.n, a subscription's meta data, like recurring total, billing period etc. were stored against the line item on the original order for that subscription.

In v2.0, this data was moved to a distinct subscription object which had its own line items for those amounts. This function bridges the two data structures to support deprecated functions used to retreive a subscription's meta data from the original order rather than the subscription itself.

Parameters
$order : WC_Order

A WC_Order object

$product_id : int = ''

The product/post ID of a subscription

Tags
since
2.0
Return values
null|object — A subscription from the order, either with an item to the product ID (if any) or just the first subscription purchase in the order.
